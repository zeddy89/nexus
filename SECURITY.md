# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Nexus, please report it responsibly by emailing:

**security@example.com**

Please do **not** open a public GitHub issue for security vulnerabilities.

### What to Include

When reporting a vulnerability, please include:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fixes (optional)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 7 days
- **Resolution Timeline**: Depends on severity and complexity

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.x.x   | :white_check_mark: |

## Security Best Practices

When using Nexus:

- Store sensitive credentials in environment variables or secure vaults
- Use SSH keys instead of passwords when possible
- Limit inventory access to authorized users
- Review playbooks before execution in production environments
